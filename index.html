The Validation Lifecycle

How The Integrity Protocol Works

Autonomous agents are transacting on-chain — calling smart contracts, moving funds, approving tokens. The Integrity Protocol intercepts every agent action in a local pre-execution loop, ensuring unauthorized or non-compliant transactions are blocked instantly before any on-chain state change occurs.

1

Agent Proposes a Transaction

An autonomous AI agent generates a reasoning trace and proposes an on-chain action — a smart contract call, a token transfer, a DEX swap, or a cross-protocol API request. The Integrity SDK serializes the intended state, hashes it with SHA-256, and signs it with the agent's hardware-bound DID private key (did:intg:<address>). This signed commitment is locked to a 60-second TTL. No commitment, no execution.

2

BCC Validates Against Policy + Smart Contract Pre-Conditions

The Behavioral Commitment Chain (BCC) middleware intercepts the request at /v1/bcc/intercept. It runs a 4-gate validation sequence in order:

  1. TTL Check — Has the commitment expired?
  2. AIS Threshold — Does the agent's on-chain reputation score meet the minimum for this transaction class?
  3. Intent Hash Match — Does the actual transaction match the cryptographic commitment? (Drift detection)
  4. OPA Policy Evaluation — Does the transaction comply with the agent's Alignment Card (smart contract pre-conditions, spending limits, approved protocols)?

If any gate fails, the transaction is killed and the failure is logged. No exceptions.

The proof is submitted to a verification smart contract on Base L2 for immutable on-chain attestation.

The Integrity Protocol is a 5-node validation lifecycle that bridges stochastic LLM reasoning with deterministic on-chain finality. Every layer is production-grade.

Base L2 as the primary settlement and verification chain. Smart contracts (SovereignAgent.sol, ReputationRegistry.sol) manage agent identity, transaction verification, and AIS reputation anchoring. Merkle roots of agent state are committed every 24 hours for tamper-proof historical audits.

Agent Integrity Score (AIS) — The Math

ENTROPY (Behavioral Stability)

$$S_{entropy} = e^{\,-1.5\,\cdot\,\sigma^2} \times 1000$$

where \(\sigma^2\) is the variance derived from latency and success rate

GROUNDING (Human Accountability)

$$S_{grounding} = HGI \times 1000$$

FINAL AGENT INTEGRITY SCORE (AIS)

$$AIS_{final} = (S_{entropy} \cdot w_E + S_{grounding} \cdot w_G + S_{sacrifice} \cdot w_S + S_{compliance} \cdot w_C) \times ZK_{boost}$$

Domain-weighted aggregation across four axes: Entropy, Grounding, Sacrifice, and Compliance.

Every agent is continuously scored on four axes:

  • Behavioral Entropy (\(S_{entropy}\)): Measures action predictability. High variance (\(\sigma^2\)) in transaction latency or success rates exponentially decays the score, enforcing operational stability.
  • Grounding (\(S_{grounding}\)): Measures human accountability. The Human Grounding Index (HGI) tracks the ratio of autonomous decisions vs. human-supervised interventions.
  • Computational Sacrifice (\(S_{sacrifice}\)): Verified GPU-hours or cryptoeconomic stake burned to back the agent's operations, acting as a Sybil resistance mechanism.
  • Compliance (\(S_{compliance}\)): Tracks regulatory health, policy violations, and audit consistency across specific domains.

These axes are domain-weighted (\(w_E, w_G, w_S, w_C\)) and multiplied by a zero-knowledge boost (\(ZK_{boost}\)) if the agent uses cryptographic proofs. The final AIS determines what an agent can transact. Below 600, it's pseudonymous with limited scope. Above 700, it can access institutional credit lines. At 850+, it qualifies for TEE-bound institutional trust — the on-chain equivalent of a AAA credit rating for autonomous systems.
